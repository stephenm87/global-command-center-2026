The Big Idea: Teachers as Architects, AI as Builder

You don't need to learn to code. You need to learn to describe what you want.

Modern AI coding assistants can write, debug, and deploy entire web applications from natural-language descriptions. Your role as a teacher is the most important one: you provide the pedagogical vision ‚Äî what students should see, explore, analyze, and create. The AI handles the syntax, the libraries, and the build pipeline.

This guide walks you through the complete process of turning an idea into a live, interactive web tool that your students can access from any device.

‚ÑπÔ∏è What You'll End Up With A live website, hosted for free, that auto-updates every time you push new code ‚Äî accessible to every student and parent via a simple URL.

Phase 1 ‚Äî Setting Up Antigravity

Antigravity is the AI-powered coding environment where all the magic happens. Think of it as a "mission control" for building software ‚Äî you type what you want in natural language; the AI writes and runs the code.

What's Inside Antigravity

  • A code editor ‚Äî where your project files live
  • A terminal ‚Äî for running build commands, installing packages, and deploying
  • A browser ‚Äî so the AI can visually verify what it built
  • MCP servers ‚Äî plug-in connections to external services (see below)

MCP Servers: Extending AI's Reach

MCP (Model Context Protocol) servers let the AI assistant access external tools and APIs. They're configured in a mcp_config.json file:

Server Purpose API Key Required?
GitHub Version control ‚Äî push code, create repos, manage files Yes ‚Äî Personal Access Token
Serper Google search API ‚Äî fetches real-time news headlines and links Yes ‚Äî serper.dev (2,500 free searches)
Firecrawl Web scraping ‚Äî verifies links and extracts content Yes ‚Äî firecrawl.dev (500 free pages)
NotebookLM Google's research tool ‚Äî query your uploaded sources for citation-backed answers Google account authentication
‚ö†Ô∏è API Keys = Small Costs (Mostly Free) Most APIs offer free tiers sufficient for educational use. GitHub and NotebookLM are free. Monitor your usage on API dashboards.

The News Search Protocol

If your project includes live news or current events, here's the pipeline:

News Search ‚Üí Verified Intel Pipeline
1

Search via Serper API

AI runs Google searches for current news on your topic (e.g., "Sudan ceasefire 2026").

2

Scrape & Verify via Firecrawl

Candidate links are scraped to verify they're accessible and non-paywalled. Broken links are discarded.

3

Curate into JSON

Verified links become a live_intel.json file with structured fields (title, source, URL, coordinates).

4

Display in Your App

The app fetches the JSON and renders it ‚Äî as sidebar cards, map pins, or news tickers.

üí° No API Needed for Static Data If your project uses historical data or teacher-curated material (not live news), you don't need Serper or Firecrawl. Just create your data as a CSV or JSON file in the public/ folder.

Phase 2 ‚Äî Scaffolding Your Project

Every project starts the same way: create a folder, install tools, start a local server.

Install Once (Before Your First Project)
  • Node.js (v18+) ‚Äî nodejs.org
  • Git ‚Äî git-scm.com (may already be installed on macOS)
  • GitHub Account ‚Äî github.com/signup
  • Netlify Account ‚Äî app.netlify.com (sign up with GitHub)

Create Your Project

# 1. Create the project folder
mkdir my-awesome-project && cd my-awesome-project

# 2. Scaffold a React + Vite app
npx -y create-vite@latest ./ -- --template react

# 3. Install dependencies & start dev server
npm install && npm run dev

Open http://localhost:5173 ‚Äî you'll see a blank React page ready to customize.

Phase 3 ‚Äî Building with AI Prompts

This is the creative core. You describe what you want; the AI writes the code. The quality of your prompts determines the quality of the result.

The Prompt Formula

Effective prompts have three layers: Structure (layout, components), Content (what data to show, where it comes from), and Aesthetics (visual style, animations, fonts).

üìå Example First Prompt "Build a full-screen dashboard with a 3D globe in the center showing 85 geo-coded events from a CSV file. Add a sidebar on the right showing a scrollable Intel Feed with filter dropdown. Use a dark cyberpunk theme with Orbitron font and neon cyan accents. Bottom bar should show key metrics: Total Events, Gold Forecast, Global Growth, and a tension meter."

Initial vs. Iteration Prompts

Initial prompt ‚Äî Long and detailed. Sets the vision for the entire project.

Iteration prompts ‚Äî Short and targeted. Change one thing at a time:

  • "Add a filter dropdown for the categories"
  • "Make that animation slower ‚Äî 1 second to 4 seconds"
  • "When I click an event, show it through 7 different theoretical perspectives"
  • "Remove the student pin feature entirely"
üí° The Golden Rule of Iteration Change one thing, verify it works, then change the next thing. If you ask for 10 changes at once and something breaks, you won't know which change caused it.

Phase 4 ‚Äî Connecting Your Data & APIs

Every educational project needs data. Two options:

Approach Best For How
Static files (CSV / JSON) Historical data, teacher-curated content, quiz questions Place files in public/ folder ‚Äî Vite copies them to build output automatically
Live APIs (Serper + Firecrawl) Current events, live scores, real-time news Search ‚Üí scrape ‚Üí verify ‚Üí save as JSON (see Phase 1 pipeline)
‚ö†Ô∏è API Keys Are Sensitive Never commit API keys to your public GitHub repository. For client-side apps, run the API search locally, save results to a JSON file, and commit the JSON (not the key). For server-side apps, use environment variables.

Verifying Your Sources

  • Test every URL in an incognito browser window
  • Reject paywalled sources ‚Äî students can't access them
  • Prefer official sources (UN, WHO, government sites, .org/.edu)
  • Re-verify periodically ‚Äî news links go stale

Phase 5 ‚Äî GitHub + Netlify: Go Live for Free

GitHub stores your code online. Netlify hosts it as a live website. Together, they give you a one-command deployment pipeline: push code ‚Üí site auto-updates.

First-Time Setup

# 1. Create a repo at github.com/new (set to Public)

# 2. Initialize and push
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR-USERNAME/project-name.git
git branch -M main
git push -u origin main

# 3. On Netlify: Add new site ‚Üí Import from GitHub
#    Build command: npm run build
#    Publish directory: dist

Daily Workflow (3 Commands)

git add .                                    # Stage changes
git commit -m "Describe what you changed"    # Save snapshot
git push origin main                         # Push ‚Üí auto-deploys

Frequently Asked Questions

Do I need to learn to code?

No. The AI writes the code. You need to describe what you want clearly. Over time, you'll naturally start to understand patterns ‚Äî but it's not required.

Is this really free?

The core stack is free: GitHub, Netlify (education plan), Node.js, React, Vite. The AI assistant (e.g., Claude Code, Cursor) requires a subscription. API keys for live news have free tiers sufficient for educational use.

Can students contribute?

Yes! Add them as GitHub collaborators. They can fork, modify, and submit pull requests ‚Äî a real-world skill.

How do I update content after launch?

Edit your data files (CSV/JSON), commit, and push. For bigger changes, open a new Antigravity conversation.
